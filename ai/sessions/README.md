# ai/sessions/

Session logs — one file per significant feature build.

Committed with the PR so reasoning lives alongside the code in git history.

## Naming convention

`YYYY-MM-DD-<branch-name>.md`

Example: `2026-03-26-feat-user-auth.md`

## When to create one

- Any feature that took more than one retry to get right
- Any session where something went wrong or confused Claude
- Any session where an architecture decision was made or reconsidered
- Not needed for: small fixes, docs updates, dependency bumps
